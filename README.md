## Who this is for

You already formalize mathematics in Lean and are comfortable writing
`theorem … := by …`, but you have **never written meta-code** and don't assume any
**functional-programming** background (monads, `do`-notation, pattern matching).
Chapter 0 starts exactly there. It teaches the small slice of functional Lean you
need, and the chapters build up to writing real tactics, ending with a
from-scratch backtracking propositional prover, `mytauto`.

By the end you can both **read** the source of Lean's and Mathlib's tactics and
**write** your own.

## Layout

```
LeanMeta.lean              table of contents (imports everything; `lake build` checks it all)
LeanMeta/
  C0_Orientation.lean      what metaprogramming is; the four monads; a Lean primer (§0.4)
  C1_ExprAndName.lean      `Name` and `Expr`, the kernel's language
  C2_MetaM.lean            `MetaM`; a goal is a metavariable
  C3_Syntax.lean           `Syntax`, quotations, macros, hygiene
  C4_Elaboration.lean      elaboration: `Syntax → Expr`
  C5_TacticM.lean          writing real tactics, step by step
  C6_Capstone.lean         `mytauto`, a backtracking propositional prover
  C7_NextSteps.lean        debugging, cheat sheet, glossary, next steps
  Exercises/C0.lean … C6.lean   problems to solve (write your answers here)
  Solutions/C0.lean … C6.lean   worked, compiling answers (peek after you try)
```

Read the chapters in order. For each chapter, try `Exercises/Cn.lean`, then check
`Solutions/Cn.lean`. The solutions double as a regression test suite; they are part
of `lake build`.

## How to use it

1. Open this folder in VS Code (with the Lean 4 extension installed), or launch a
   Codespace with the badge above.
2. Open `LeanMeta/C0_Orientation.lean` and start reading.
3. Step through each chapter top to bottom, watching the **Infoview** on the right.
   Put your cursor on each `#eval` / `#check` / `#print` / `example` and read what it
   shows. The files are designed to be *looked at*, not just read.

No dependencies beyond Lean's own library (`import Lean`); no Mathlib needed.
